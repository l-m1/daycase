#  

.字符可以代表任意单个字符 两种情况除外  
        - UTF-16 可以加u进行解决  
        - 行终止符(\n、\r、行分隔符、段分隔符) 使用s解决  
        - .可以匹配任意单个字符 即 dotAll模式  
^匹配行行首$匹配行尾  
正则的扩展:  
        - es6修饰符 第二个参数修饰符将原本正则表达式中的修饰符替换掉 使用flags  
        - 字符串的正则方法  
            - match()、replace()、search()、split()  
            - String.prototype.match 调用 RegExp.prototype[Symbol.match] 其它同理  
        - u修饰符 用来处理编码大于0FFFF的字符  
            - 除了换行符以外，其它任意码点大于0FFFF的字符，点字符不能识别，必须加上u  
            - Unicode 必须加上u 才能识别大括号 否则为量词  
            - 量词加u之后，都能正确识别码点大于0FFFF的字符  
            - 预定义模式  
            - i修饰符 加u修饰符可识别规范字符
            - 转义 没u前面的\无效 有u则报错  
        - Y修饰符 粘连修饰符  
            - y与g类似，都是全局匹配  
                - g没有位置要求 上一次匹配结束 继续匹配剩下符合要求的  
                - y有位置求 上一次匹配结束，下一次匹配从上一次结束位置开始，没有符合要求的则为false  
                - g与y都与lastIndex有关系 g从指定位置向后搜索 y从指定位置进行匹配  
                - y只匹配一个，g匹配所有  
                - g修饰符会忽略非法字符 y不会  
            - y相当于隐藏了头部^  
        - repalce方法  
        - flags  
            - 返回修饰符  
            - resource返回要匹配的内容  
        - s修饰符 dotAll模式
